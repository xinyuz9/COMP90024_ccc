## 1. 基础入口：`with open()`

`with` 语句被称为“上下文管理器”，是处理文件最专业、最安全的方式。

- **核心定义**：自动管理文件的打开与关闭。
    
- **优势**：即便在处理 JSON 出现 `JSONDecodeError` 或程序崩溃时，它也能确保文件被正确关闭，释放系统资源。
    
- **HPC 关联**：在并行读取大文件（如 `mastodon-large.ndjson`）时，每个进程都会独立打开一个文件句柄 。
    
```Python
with open("file.ndjson", "r", encoding="utf-8") as f:
    # 所有的操作都在这个缩进块内完成
    data = f.readline()
# 缩进结束后，文件自动关闭
```

## 2. 定位指针：`f.seek()` 与 `f.tell()`

这两个函数就像文件的“GPS”，在 **Chunking (分块)** 方案中不可或缺。

### `f.tell()`

- **作用**：返回文件指针当前所在的 **字节位置 (Byte Offset)**。
    
- **用途**：在并行处理中，用来判断当前进程是否已经读到了属于自己的 `end_byte` 边界。
    

### `f.seek(offset, whence)`

- **作用**：将文件指针移动到指定的字节位置。
    
- **参数**：
    
    - `offset`: 移动多少个字节。
        
    - `whence`: 可选，0 代表从文件头开始（最常用），1 代表从当前位置，2 代表从文件末尾。
        
- **HPC 场景**：每个 MPI 进程计算出自己的 `start_byte` 后，第一时间调用 `f.seek(start_byte)` 开启自己的任务区间。
    

---

## 3. 读取内容：`f.readline()` 与 `f.read()`

### `f.readline()`

- **作用**：从当前位置读取直到遇见换行符 `\n` 为止的一整行。
    
- **断头行处理**：
    
    - 在 Chunking 中，由于 `seek` 可能落在行中间，第一个 `readline()` 常用于丢弃残缺的行（由前一个进程处理）。
        
    - 它是处理 `.ndjson` 格式（每行一个 JSON）的标准方式。
        

### `f.read(size)`

- **作用**：读取指定大小的字节。如果不传参数，则读取整个文件（**警告：** 绝对不要对 `large.ndjson` 这样做，会导致内存溢出）。
    

---

## 4. 辅助函数：`os.path.getsize()`

- **作用**：获取文件的总字节大小。
    
- **必要性**：在 `solution.py` 中，你需要先获取文件总大小，才能用地板除 `//` 来计算每个进程的 `chunk_size`。

---

## `f.readlines(hint)`

这是处理大规模 `.ndjson` 数据的利器，能显著减少系统调用（System Calls）。

- **功能**：读取文件中的多行并返回一个 **字符串列表 (List of Strings)**。
    
- **参数 `hint`**：
    
    - **作用**：指定读取的**近似字节数**。
        
    - **逻辑**：Python 会持续读取完整的行，直到总字节数达到或超过 `hint`。它**绝不会**在行中间截断，保证了每一行 JSON 的完整性。
        
- **返回值**：
    
    - **正常**：包含多行数据的 `list`。
        
    - **EOF (文件末尾)**：返回一个空列表 `[]`（**注意：不是 `None`**）。
